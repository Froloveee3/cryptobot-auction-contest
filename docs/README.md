# Документация

Проект — система многораундовых аукционов (Telegram Gift Auctions‑like) с real‑time обновлениями, анти‑снайпингом и ботами.

## Ссылки на действующий стенд

- **Сайт (Frontend)**: `https://www.cryptobot-auction-contest.ru` (также доступен `https://cryptobot-auction-contest.ru`)
- **API**: `https://api.cryptobot-auction-contest.ru/api`
- **WS**: `https://ws.cryptobot-auction-contest.ru` (Socket.IO namespace: `/auctions`)
- **Видео‑демо**: (вставить ссылку)

## Архитектура (в одном абзаце)

Backend (NestJS) хранит данные в MongoDB (replica set нужен для транзакций), использует Redis + BullMQ для фоновых задач и Socket.IO для real‑time.
Frontend (React/TS) ходит в REST API и подписывается на события Socket.IO `/auctions`.

## Ключевая логика

- **Аукцион состоит из раундов** (`totalRounds`). В каждом раунде ограничено время и есть leaderboard (Top‑100).
- **Подарки/призы** распределяются по раундам: в UI показывается прогресс `x/y` (сколько уже разыграно).
- **Одна активная ставка на пользователя на аукцион**. Если ставка уже существует — используется `raise`, иначе `new`.
- **Антиснайпинг**: ставка в конце раунда может продлить раунд (по правилам окна/продления).
- **Боты**:
  - учётки ботов используют зарезервированный префикс `_bot…`,
  - боты избегают ставок в “последнюю секунду”, чтобы не спамить продлениями.

## Как запустить локально (Docker)

1) Подготовить env:

- `.env.example` → `./.env`

2) Запустить dev‑стек:

```bash
docker compose --profile dev up -d mongodb redis mongodb-init backend backend-worker frontend
```

3) Открыть:

- **Frontend**: `http://localhost:3001`
- **API**: `http://localhost:3000/api`
- **Swagger**: `http://localhost:3000/api/docs`

Остановить:

```bash
docker compose --profile dev down
```

## Ручная проверка через UI (быстрый сценарий)

1) Зайти на сайт, зарегистрировать 2 пользователей (без `_` в username).
2) На Dashboard пополнить баланс обоим пользователям.
3) Создать аукцион (желательно включить ботов и указать количество).
4) Перейти в аукцион:
   - видно прогресс подарков `x/y`,
   - видно баланс в шапке,
   - ставка делается/повышается (UI сам выбирает `new` или `raise`),
   - при ставках в конце раунда (в окне анти‑снайпинга) может быть продление.
5) Проверить **Профиль**:
   - вкладка **Коллекция** (выигранные подарки),
   - **История ставок**,
   - **История баланса** (ledger).

## Postman: ручное тестирование API

В `docs/` лежат готовые файлы:

- **Collection**: `docs/postman-collection.json`
- **Environment**: `docs/postman-environment.json`

Как использовать:

1) Import → выбрать оба файла.
2) В Environment поменять `baseUrl` на один из вариантов:
   - локально: `http://localhost:3000/api`
   - прод: `https://api.cryptobot-auction-contest.ru/api`
3) Запускать запросы по порядку (папки 00→05).

### Что покрывает коллекция

- регистрация 2 пользователей, логин администратора
- депозит пользователям
- создание/старт аукциона
- ставки `new` / `raise`
- чтение истории профиля (`/users/me/*`)
- (опционально) проверка ботов (admin endpoints)

## Полезные факты / допущения

- **Username (web)**: только `^[A-Za-z][A-Za-z0-9]*$` (без `_`). Префикс `_bot` зарезервирован под ботов.
- **JWT**: есть авто‑refresh токена, чтобы сессия не “отваливалась” пока вы на сайте.
