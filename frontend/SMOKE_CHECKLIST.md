# Frontend Smoke Checklist

Краткий чек-лист для проверки основных функций фронтенда после рефакторинга. Основан на `docs/scenarios`.

## ✅ Authentication

### Web Auth
- [ ] Регистрация нового пользователя работает
- [ ] Логин существующего пользователя работает
- [ ] Отклонение неверных credentials работает
- [ ] JWT токен сохраняется в localStorage
- [ ] Автоматический редирект после успешного auth

### Telegram WebView Auth
- [ ] Автоматический логин через Telegram initData работает
- [ ] Показывается "Connecting..." экран в Telegram WebView
- [ ] initData сохраняется в sessionStorage (не localStorage)
- [ ] Telegram WebApp.ready() и expand() вызываются
- [ ] Обработка ошибок (нет bot token, устаревший initData)

## ✅ Lobby (Dashboard)

### Загрузка и отображение
- [ ] Список аукционов загружается при открытии страницы
- [ ] Фильтры работают (Active / All / History)
- [ ] Пагинация работает ("Load More" загружает следующую страницу)
- [ ] Адаптивный layout (1 колонка на mobile, 2-3 на desktop)
- [ ] Карточки аукционов отображают корректную информацию

### Real-time обновления
- [ ] Новый аукцион появляется в списке через WS `auction:created`
- [ ] Новый аукцион добавляется только если соответствует текущему фильтру
- [ ] Нет дубликатов при добавлении через WS

## ✅ Auction Page

### Загрузка и инициализация
- [ ] Аукцион загружается через HTTP при открытии страницы
- [ ] WS snapshot приходит и инициализирует состояние
- [ ] Top 100 отображается корректно
- [ ] Позиция пользователя (myRank) отображается
- [ ] Текущий раунд отображается с корректным таймером

### Real-time обновления
- [ ] `auction:patch` обновляет состояние инкрементально
- [ ] Top 100 обновляется через patch
- [ ] `dynamicMinBid` обновляется в реальном времени
- [ ] `remainingSupply` обновляется в реальном времени
- [ ] Позиция пользователя обновляется через patch
- [ ] Нет polling (HTTP запросы каждые 5s)

### Bid Form
- [ ] Режим "New" работает (создает новую ставку)
- [ ] Режим "Raise" работает (увеличивает существующую ставку)
- [ ] Валидация: `amount >= dynamicMinBid` для new
- [ ] Валидация: `delta >= minIncrement` для raise
- [ ] Toast уведомление при успешной ставке
- [ ] Toast уведомление при ошибке (BID_TOO_LOW, INSUFFICIENT_BALANCE, etc.)

### Round Events
- [ ] `round:started` обновляет UI и показывает toast
- [ ] `round:ended` обновляет UI и показывает toast
- [ ] `round:extended` обновляет UI и показывает toast
- [ ] Таймер корректно отображает оставшееся время
- [ ] Завершение раунда корректно отображается в UI

### Displacement
- [ ] Toast показывается когда пользователя выбили из топа
- [ ] Toast содержит понятное сообщение
- [ ] `displacedUserIds` из `bid:placed` корректно обрабатывается

## ✅ Toast Notifications

- [ ] Toast появляется в правом верхнем углу
- [ ] Toast автоматически исчезает через заданное время
- [ ] Toast можно закрыть вручную (кнопка ×)
- [ ] Разные типы toast (success/error/info/warning) имеют разные цвета
- [ ] Анимация появления/исчезновения работает плавно

## ✅ WebSocket Connection

### Connection Management
- [ ] WS подключается при открытии страницы
- [ ] WS переподключается при разрыве соединения
- [ ] Комнаты (lobby/auction) автоматически переподключаются после reconnect
- [ ] Heartbeat работает (app:ping/app:pong)

### Authentication
- [ ] JWT токен передается в join:auction для web auth
- [ ] Telegram initData передается в join:auction для Telegram auth
- [ ] Auth обновляется при смене пользователя

## ✅ Error Handling

- [ ] Ошибки API показываются через toast (не alert)
- [ ] Специфичные error codes обрабатываются корректно
- [ ] Fallback сообщения показываются для неизвестных ошибок
- [ ] Нет console.error в production (только в dev)

## ✅ Adaptive Design

- [ ] Mobile layout (<768px): 1 колонка для карточек
- [ ] Desktop layout (>=768px): 2-3 колонки для карточек
- [ ] Все формы адаптивны к размеру экрана
- [ ] Навигация работает на мобильных устройствах

## ✅ Performance

- [ ] Нет лишних HTTP запросов (polling убран)
- [ ] WS события обрабатываются эффективно
- [ ] Нет memory leaks (компоненты корректно очищаются)
- [ ] Сборка проходит без ошибок

## ✅ Code Quality

- [ ] Нет неиспользуемых методов в сервисах
- [ ] Боты скрыты от обычных пользователей
- [ ] Все console.log обернуты в dev проверки
- [ ] TypeScript типы корректны
- [ ] ESLint ошибок нет

## Быстрая проверка (5 минут)

1. Открыть Dashboard → проверить фильтры и пагинацию
2. Создать аукцион → проверить что он появился в списке через WS
3. Открыть аукцион → проверить что snapshot приходит
4. Поставить ставку → проверить toast и обновление через patch
5. Проверить что нет polling в Network tab
